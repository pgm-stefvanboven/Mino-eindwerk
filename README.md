## 1. Projectomschrijving

**Mino** is een interactief zorgondersteunend systeem dat focust op medicatiebegeleiding, dagstructuur en geruststelling voor gebruikers die extra ondersteuning nodig hebben in hun dagelijkse routine.

Het project bestaat uit:

* een mobiele applicatie (frontend),
* een backend met logica en data-afhandeling,
* en een fysieke component (robot / device) die signalen en interacties ondersteunt.

De kern van Mino draait rond één centrale interactieflow:
**ondersteuning → handeling → bevestiging → geruststelling**.

Het eindresultaat is een **functioneel prototype** dat aantoont hoe technologie kan bijdragen aan een gevoel van structuur en vertrouwen binnen een zorgcontext.

## 4. Prototype-context

Dit project werd ontwikkeld als een **afstudeerprototype**.
De volledige werking van Mino komt pas tot zijn recht in combinatie met de fysieke robot, die **niet kan worden meegeleverd** bij deze indiening.

Om die reden:

* is er **geen login of testaccount** voorzien,
* is de applicatie **niet bedoeld voor zelfstandig gebruik door de jury**,
* wordt de werking verduidelijkt via visuals (screenshots / video).

De focus ligt op **concept, interactielogica en ervaring**, niet op een kant-en-klare productdemo.

---

## 5. Bekende beperkingen

* Dit project is een **prototype**, geen productieklare zorgapplicatie
* Hardware-interacties zijn deels gesimuleerd of contextafhankelijk
* De applicatie is ontworpen als onderdeel van een **ecosysteem**, niet als standalone product
* Er wordt geen medische correctheid of validatie geclaimd
